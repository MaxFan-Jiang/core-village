先別緊張

GitHub，其實就是一間
大家共用的「數位工作室」

你可能聽過「工程師都在 GitHub 上」，覺得它很硬。其實把它想成一間工作室就好：每個人都能進來、看見彼此正在做的東西、把自己的那一塊放上去，而且每一次改動都被完整記錄，永遠不會弄丟。

📦Repository

倉庫

一個專案的資料夾。我們的村莊就住在一個倉庫裡，大家的作品都放在這。

🍴Fork

分叉

把這個倉庫複製一份到你自己的帳號，變成你的遊樂場，怎麼改都不會動到別人。

💾Commit

提交

一次存檔。像遊戲存檔點，記下「我這一步改了什麼」。

📬Pull Request

拉取請求

「我改好了，請看看要不要收進去。」一封禮貌的提案信，簡稱 PR。

記住這一句

Fork 是「把專案複製一份到你家」，PR 是「我在我家改好了，請你看要不要收進去」。
整個 GitHub 協作，就是這兩個動作在來回。

五個關鍵字

看到這些字，不要怕

整份教學只會用到這五個詞。先用生活化的方式記起來，等下實際操作時你就會發現：原來就這樣。

📦 Repository 倉庫＝一個專案的資料夾
🍴 Fork 分叉＝把別人的專案複製一份到你帳號
✏️ Commit 提交＝存一次檔，並寫下「這次改了什麼」
🌿 Branch 分支＝一條平行的草稿線，改壞了也不影響正式版
📬 Pull Request 拉取請求＝把你的改動，提案給原專案收編

📌 好消息：用網頁版操作時，Branch 這一步 GitHub 會自動幫你開，你幾乎不用管它。所以真正要動手的，只有 Fork、Commit、Pull Request 三個。

打開註冊頁

💡 Username 是公開的，別人會看到。建議用英文，例如你的英文名字，乾淨好記就好。

GitHub 會寄一封信到你的 Email，裡面有一組驗證碼或一個按鈕，點一下確認「這個信箱是你的」。

  • （建議）上傳一張大頭貼，村莊裡比較好認你

STEP ③ · 重頭戲

你的第一個貢獻
全程用滑鼠，不碰終端機

我們的目標很單純：在村莊裡，放進「你的村民」。你只要新增一個小檔案，填上你的名字和幾個欄位就行。GitHub 全程在瀏覽器裡完成。

1

按下 Fork複製到你家

在村莊倉庫頁面右上角按 🍴 Fork。GitHub 會把整個專案複製一份到你的帳號，網址會變成 github.com/你的名字/core-village。這份是你的，怎麼改都安全。

2

複製範本檔

進到 villagers/，打開範本 villagers/_example.json，把裡面的內容複製一份，在 villagers/ 底下新增一個用你代號命名的檔，例如 villagers/pixel-cat.json。

💡 在 GitHub 網頁裡，建立檔案時把檔名直接打成 villagers/你的代號.json，GitHub 就會幫你把檔放到對的位置。不用先在電腦上建。
3

編輯你的小檔案

點開檔案右上角的鉛筆 ✏️，把內容改成你的。就這麼幾行：

villagers/pixel-cat.json
{
"id": "pixel-cat",
"name": "像素貓",
"job": "自由創作 · 來蓋村民",
"emoji": "🐱",
"hp": 90, "atk": 20, "spd": 14,
"skill": "dodge",
"cry": "我閃，你打不到我。"
}
4

提交改動（Commit）＝存檔

改完往下找 Commit changes。在訊息欄寫一句白話的「我做了什麼」，例如 加入像素貓這個村民，按綠色按鈕。

💡 Commit 訊息就是寫給人看的存檔備註，用中文、講人話都可以。
5

開 Pull Request（PR）送出提案

回到你的 fork 頁面，GitHub 通常會跳出一條黃色提示，寫著 Compare & pull request，點它。確認一下要送的內容，按 Create pull request。

送出之後，這封「提案信」就到了村長（Max）手上。他看過、按下 Merge，你的村民就正式進到公開村莊裡了。🎉

github.com/你的名字/core-village
🌿 你剛剛推了一個改動到 villagers/pixel-cat.json Compare & pull request
點綠色按鈕 → 下一頁按 Create pull request → 完成 ✅
整段濃縮成一句

Fork 一份 → 在 villagers/ 加一個你的 .json → 改好欄位 → Commit → 開 PR。
五步，全部在瀏覽器裡，滑鼠就能做完。

卡關自救

遇到這些，別慌

新手最常卡的幾關，先在這裡幫你準備好答案。GitHub 的好處是：幾乎沒有什麼是救不回來的。

Q我把檔案改壞了，怎麼辦？+
完全不用怕。你動的是你 fork 裡的複本，原本的村莊一個字都沒被改到。真的亂了，最簡單的做法：把你的 fork 刪掉，重新 Fork 一次，等於重新發一份乾淨的複本。你的每一次 Commit 也都留著記錄，隨時能回到上一個存檔點。
Q看到「merge conflict」這個紅字，是不是我搞砸了？+
不是。它只是說「同一行，兩個人都改了，GitHub 不知道該聽誰的」。因為我們刻意讓每個人只動自己那一個檔，所以你幾乎不會遇到。萬一真的遇到，直接在 PR 留言 tag 村長，或丟到 LINE 群，30 秒幫你解掉。
Q忘記 GitHub 密碼了？+
登入頁點 Forgot password，GitHub 會寄重設信到你註冊的 Email。順帶一提，第一次操作時 GitHub 可能會要你做一次手機或 Email 的兩步驟驗證，那是保護你帳號的，照著做就好。
Q我的 PR 被回覆「要再改一下」，是被拒絕了嗎？+
不是。這在工程師世界裡是最正常的日常，叫 code review。村長或夥伴給你一點建議，你回到自己的 fork 再改、再 Commit，PR 會自動更新，不用重開。來回幾次，作品會更好。被 review 不是被否定，是被認真對待。
Q別人後來更新了村莊，我的 fork 會不會太舊？+
會，但一鍵就能追上。到你 fork 的頁面，會看到一個 Sync fork 按鈕，點它，你的複本就跟最新的村莊同步了。建議每次要新增村民前，先按一下。
Q我真的零基礎，會不會太難？+
這份教學就是為零基礎寫的。我們村裡有老師、有插畫家、有業務、有機構工程師，大部分人都沒寫過程式。第一個 PR 只是改幾行字、填個名字。你做得到，而且做完那一刻會很有感。卡住了，LINE 群隨時喊一聲。
